### 4.2 개별 경쟁사 상세 분석

#### Brave Rewards (BAT)
- **설립**: 2016년 (Brendan Eich, Mozilla/Firefox 공동창립자)
- **규모**: 6,500만+ MAU, BAT 시가총액 $300M+
- **작동 방식**: 브라우저 내 옵트인 광고 → 광고 수익의 70%를 BAT로 유저에게
- **핵심 강점**: 검증된 비즈니스 모델, 대규모 유저 베이스
- **핵심 약점**: 브라우저 전환 필요, 광고당 보상이 매우 적음 (~$0.01-0.05)

#### Permission.io (ASK)
- **투자**: $50M 이상 (equity + token sale)
- **규모**: 450,000+ 유저 (2020년 기준)
- **작동 방식**: 크롬 확장으로 광고 시청 → ASK 토큰 획득
- **핵심 강점**: 데이터 주권 내러티브, AI 어시스턴트 통합
- **핵심 약점**: 유저 규모 작음, 토큰 유동성 낮음

#### Sweatcoin
- **규모**: 1.4억 등록 유저, 2023년 $20M 매출
- **비즈니스 모델**:
  - 리워드 광고 (Daily Rewards)
  - 600+ 브랜드 마켓플레이스
  - 프리미엄 구독 $4.99/월
- **핵심 강점**: 건강과 보상 결합, 비크립토 유저도 접근 가능
- **핵심 약점**: 봇/가짜 걸음 문제, Move-to-Earn Ponzi 논란

#### Galxe
- **규모**: 3,200만 유저, 6,700+ 파트너
- **작동 방식**: Web3 퀘스트 완료 → 토큰/NFT/에어드랍 자격
- **핵심 강점**: B2B 모델로 안정적 수익, 대규모 파트너 네트워크
- **핵심 약점**: 에어드랍 헌터 위주, 일반인 진입장벽 높음

#### Layer3
- **규모**: 720만 유저, $15M Series A
- **작동 방식**: 온체인 퀘스트 + 교육 → XP/토큰 획득
- **핵심 강점**: 게이미피케이션, 교육 콘텐츠 결합
- **핵심 약점**: Galxe와 유사하게 Web3 네이티브만 타겟

## 5. 시장 Gap 분석

### 5.1 해결되지 않은 문제들

| Gap | 현황 | 영향 |
|-----|------|------|
| **봇/Sybil 문제** | Permission, Cointiply 등은 봇 남용에 취약. Brave도 완벽히 해결 못함 | 광고주 ROI 저하, 플랫폼 신뢰도 하락 |
| **광고 참여 품질** | 대부분 "보기만" 하면 됨. 실제 이해/기억 여부 미검증 | 광고 효과 측정 불가, CPM 대비 가치 의문 |
| **Web3 진입 장벽** | Galxe/Layer3는 이미 크립토 유저만 사용 | 시장 확장 한계, 일반인 온보딩 실패 |
| **지속가능한 가치 교환** | 많은 Watch-to-Earn이 Ponzi 구조 의심받음 | 규제 리스크, 유저 신뢰 문제 |

### 5.2 기회 영역

1. **검증된 인간 광고 시장**: World ID로 Sybil 문제 해결 시 프리미엄 CPM 가능
2. **참여 품질 메트릭**: 퀴즈 정답률 = 광고 이해도 측정 가능
3. **World App 생태계**: 300+ 미니앱, 일 45,000+ 신규 지갑 생성

## 6. Advertise 차별화 전략

### 6.1 차별화 축 1: World ID 기반 Sybil 저항성

**차별점**: 1인 1계정 보장으로 광고주에게 "검증된 실제 인간"에게만 광고 노출 약속

**광고주 가치 제안**:
- 봇 트래픽 0% 보장
- 실제 인간 도달 수 정확히 측정
- 프리미엄 CPM 정당화

**왜 기존 기업들이 안 했나?**
| 기업 | 이유 |
|------|------|
| Brave | 2016년 설립, World ID는 2023년에 등장 (기술 시점 차이) |
| Permission | 기존 인프라에 World ID 통합 비용/복잡성, Polygon 기반 vs World Chain |
| 대부분 | World ID는 Orb 필요 → 글로벌 커버리지 제한적이었음 (현재 12M+로 개선 중) |

### 6.2 차별화 축 2: 퀴즈 기반 광고 이해도 검증

**차별점**: 광고 후 퀴즈 → 정답 시 보너스 XP. 광고주에게 "광고가 실제로 이해됐다"는 메트릭 제공

**광고주 가치 제안**:
- 광고 리콜/이해도 실시간 측정
- A/B 테스트: 어떤 메시지가 더 잘 기억되는지
- CPE (Cost Per Engagement) 모델 가능

**왜 기존 기업들이 안 했나?**
| 기업 | 이유 |
|------|------|
| Brave | 브라우저 UI에 퀴즈 UX 넣기 어려움 (passive 광고 모델) |
| Sweatcoin | 걷기가 코어, 광고는 부수적 수익원일 뿐 |
| 대부분 | 유저 이탈 우려 (추가 액션 = friction) |

**우리의 해결책**: 보너스 XP (+25)로 인센티브 설계 → friction을 기회로 전환

### 6.3 차별화 축 3: World App Mini App 생태계 통합

**차별점**: World App 300+ 미니앱 생태계에 자연스럽게 통합. 별도 앱 설치 불필요

**유저 가치 제안**:
- 이미 World App 있으면 바로 사용
- 별도 회원가입/지갑 연결 불필요
- WLD로 즉시 인출 가능

**왜 기존 기업들이 안 했나?**
| 기업 | 이유 |
|------|------|
| Brave | 자체 브라우저 생태계에 lock-in, 플랫폼 종속 꺼림 |
| Permission | 독립 플랫폼으로 성장 전략 |
| Layer3/Galxe | 멀티체인이지만 World 생태계와 무관 |

**핵심 기회**: World 생태계는 아직 초기 → 광고 미니앱 영역 선점 가능
