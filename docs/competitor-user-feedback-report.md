## 1. Brave Rewards (BAT) - 사용자 불만 분석

### 핵심 불만 사항

| 문제 카테고리 | 사용자 피드백 | 빈도 |
|--------------|--------------|------|
| **보상 소실** | "지갑 연동 후 BAT가 사라졌다" "3년간 모은 BAT가 0이 됐다" | 매우 높음 |
| **낮은 보상** | "시간당 $0.01 수준" "30일 광고 봤는데 $0.34" | 높음 |
| **출금 불가** | "Uphold 인증이 안 됨" "몇 년째 출금 못함" | 높음 |
| **계정 차단** | "갑자기 flagged 처리됨" "이유 없이 보상 중단" | 중간 |
| **고객 지원 부재** | "티켓 넣어도 답 없음" "같은 답변만 반복" | 높음 |
| **KYC 모순** | "프라이버시 브라우저가 KYC 요구" | 중간 |

### 대표 사용자 코멘트

> "I've been saving rewards for maybe three years but could never cash out. Dealing with 5 ads an hour for years, simply because I hoped to make money from it, and never being able to cash it out is frustrating."
> — [Brave Community Forum](https://community.brave.app/t/brave-rewards-not-worth-the-trouble/581901)

> "The tiny payouts, unreliable system, and poor customer support create more frustration than reward."
> — [Medium Review](https://medium.com/@aaronkavungal/is-brave-rewards-worth-your-time-7489d3ec075d)

### Advertise가 배워야 할 점

| Brave의 실수 | 우리의 해결책 |
|-------------|--------------|
| 복잡한 출금 프로세스 (Uphold 필수) | World App 내 WLD로 즉시 지급, 추가 인증 불필요 |
| 불투명한 계정 차단 | World ID = 자동 인증, 차단 사유 명확화 |
| 낮은 보상 ($0.01/광고) | 퀴즈 보너스로 참여 가치 증대 (+25 XP) |
| KYC 요구 (프라이버시 모순) | World ID는 ZK Proof로 프라이버시 보장 |

## 4. Telegram Tap-to-Earn (Notcoin, Hamster Kombat) - 실패 분석

### 핵심 문제점

| 문제 | 상세 | 결과 |
|------|------|------|
| **기대 vs 현실** | 몇 달간 탭 → 에어드랍 "dust" 수준 | 대규모 사용자 이탈 |
| **지속가능성 부재** | 토큰 배포 후 플레이어 95% 감소 | Hamster Kombat: 3억 → 1,200만 |
| **투명성 부족** | 마지막에 규칙 변경, 치터 처리 | 커뮤니티 반발 |
| **게임플레이 깊이 없음** | 탭만 하면 됨 → 지루함 | 재방문율 급락 |
| **네트워크 과부하** | TON 체인 2회 다운 | 서비스 신뢰도 하락 |

### 대표 피드백

> "Several users described their HMSTR reward as 'dust' - a term often invoked to describe trivial amounts of crypto."
> — [Decrypt](https://decrypt.co/306552/hamster-kombat-token-all-time-low-game-returns)

> "Notcoin's team told Cointelegraph that tap to earn is fading fast, prompting developers to explore genres with longer player retention."
> — [BeInCrypto](https://beincrypto.com/users-telegram-games-hamster-kombat-tapswap/)

### Advertise가 배워야 할 점

| Tap-to-Earn의 실수 | 우리의 해결책 |
|-------------------|--------------|
| 탭 = 무의미한 행동 | 광고 시청 + 퀴즈 = 의미 있는 참여 |
| 토큰 가치 급락 | WLD (기존 유틸리티, 안정적 가치) |
| 일회성 에어드랍 | 지속적인 광고 수익 기반 보상 |
| 투명성 부족 | 명확한 보상 정책, 실시간 잔액 확인 |

## 5. STEPN - Move-to-Earn 포스트모템

### 타임라인

| 시점 | 이벤트 | 결과 |
|------|--------|------|
| 2022년 3월 | GMT 토큰 출시 | 시가총액 $3B |
| 2022년 4월 | 최고점 | GMT $4, 하루 $5M 순이익 |
| 2022년 5월 | 중국 퇴출 발표 | GMT 37% 급락 |
| 2022년 6월 | GST 토큰 91% 하락 | 같은 운동에 $37 → $0.72 |
| 2022년 | DAU 80% 감소 | 신규 유저 유입 중단 |
| 현재 | 생존 중 | 시가총액 $500M, Adidas 파트너십 |

### 핵심 실패 요인

#### 1. Ponzi 구조 토큰노믹스
> "As seen in every ponzi, yields come from new capital inflows — for STEPN those are new sneaker buyers. If a ponzi runs out of new capital inflows, it will collapse."
> — [Vader Research](https://defivader.medium.com/why-stepns-collapse-is-inevitable-5259a6584a98)

#### 2. 높은 진입 비용
- NFT 스니커즈 구매 필수 (수백 달러)
- 신규 유저 진입 장벽

#### 3. 외부 규제 리스크
- 중국 규제로 갑작스런 서비스 중단
- 유저 신뢰 붕괴

#### 4. 네트워크 의존성
- Solana 다운타임으로 서비스 장애

### Advertise가 배워야 할 점

| STEPN의 실수 | 우리의 해결책 |
|-------------|--------------|
| NFT 구매 필수 (진입 비용) | 무료 시작, 구매 불필요 |
| 신규 유저 의존 토큰노믹스 | 광고 수익 기반 (지속가능) |
| 자체 토큰 가치 변동 | WLD (독립적 가치) |
| 단일 국가 규제 리스크 | 글로벌 분산, World 생태계 |

| **사용자 소유권** | 플랫폼 귀속 | 유저 지갑 소유 |

│  + World ID (봇 방지)                                │
│  + 블록체인 투명성                                    │
│  + 유저 소유권 (WLD)                                 │
│  = Advertise의 기회                                  │
└─────────────────────────────────────────────────────┘
```

## 18. 광고 참여 & 행동 전환 전략

### 18.1 핵심 지표 (KPIs)

| 지표 | 정의 | 벤치마크 | 목표 |
|------|------|---------|------|
| **광고 완료율** | 끝까지 시청한 비율 | 75.8% (업계) | **90%+** |
| **퀴즈 정답률** | 퀴즈 맞춘 비율 | N/A (신규) | **70%+** |
| **D7 Retention** | 7일 후 재방문 | 20-30% | **40%+** |
| **D30 Retention** | 30일 후 재방문 | 10-15% | **25%+** |
| **일일 광고 시청 수** | 유저당 하루 광고 | 3-5개 | **3개 (타겟)** |

### 18.2 완료율 극대화 전략

#### 왜 유저가 광고를 끝까지 볼까?

```
┌─────────────────────────────────────────────────────┐
│  동기 부여 요소                                       │
├─────────────────────────────────────────────────────┤
│  1. 명확한 보상 제시    "이 광고 = 50 XP"             │
│  2. 진행률 표시         카운트다운 타이머              │
│  3. 손실 회피           "지금 나가면 보상 없음"        │
│  4. 보너스 기대         "퀴즈 정답 시 +25 XP"         │
└─────────────────────────────────────────────────────┘
```

#### 현재 구현 vs 개선 방향

| 요소 | 현재 | 개선 방향 |
|------|------|----------|
| 보상 표시 | ◯ XP 표시 | WLD 환산 가치도 표시 |
| 진행률 | ◯ 타이머 | 프로그레스 바 추가 |
| 손실 회피 | △ 암묵적 | "나가면 0 XP" 명시 |
| 보너스 | ◯ 퀴즈 +25 | 연속 정답 보너스 |

### 18.3 퀴즈를 통한 행동 전환

#### 퀴즈의 역할

```
광고 시청 → 퀴즈 정답 → 브랜드 리콜 80%↑
                    → 참여 품질 증명
                    → 광고주 가치 제안
```

#### Ad Recall 연구 결과

| 측정 방식 | 일반 광고 | 퀴즈 연동 광고 |
|----------|----------|---------------|
| Unaided Recall | 10-30% | **50%+** (예상) |
| Aided Recall | 40-60% | **80%+** |
| Brand Lift | 기준 | **+80%** (YouGov 데이터) |

#### 퀴즈 설계 원칙

| 원칙 | 설명 | 예시 |
|------|------|------|
| **핵심 메시지 기반** | 광고의 주요 포인트 | "World ID가 증명하는 것?" |
| **4지선다** | 적절한 난이도 | 너무 쉽거나 어렵지 않게 |
| **즉각 피드백** | 정답/오답 표시 | 학습 효과 |
| **보상 차등** | 정답 = 보너스 | 참여 동기 부여 |

### 18.4 지속적 참여 유도 (Retention Loop)

#### Retention 벤치마크

> "rewarded video를 첫 주에 1회라도 본 유저의 D30 Retention은 **53.2%** — 평균(12-13%)의 **4배 이상**"
> — Tapjoy 연구 (5억+ 유저)

#### 게이미피케이션 요소

| 요소 | 심리학적 기반 | 구현 방법 |
|------|-------------|----------|
| **스트릭 (연속)** | 손실 회피, 습관 형성 | "🔥 5일 연속!" |
| **일일 목표** | 달성감, 도파민 | "오늘 광고 2/3" |
| **레벨/XP** | 진전 가시화 | "Lv.3 → Lv.4" |
| **배지** | 수집 욕구, 인정 | "Quiz Master 획득!" |
| **리더보드** | 경쟁심, 소속감 | "주간 Top 10" |

#### 최적 빈도

| 지표 | 권장 | 이유 |
|------|------|------|
| 일일 광고 수 | **3-5개** | 과하면 피로, 적으면 습관 안 됨 |
| 세션당 광고 | **2-3개** | 집중력 유지 |
| 광고 간 간격 | **자연스러운 타이밍** | 강제감 제거 |

#### Retention Loop 설계

```
┌──────────────────────────────────────────────────────────────┐
│                    Daily Retention Loop                       │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│    [푸시 알림]                                                │
│        ↓                                                     │
│    "오늘 일일 목표 달성하고 +50 XP 받으세요!"                    │
│        ↓                                                     │
│    [앱 오픈] → 스트릭 확인 → 일일 목표 표시                      │
│        ↓                                                     │
│    [광고 시청] → 퀴즈 → 보상 → 진전 피드백                       │
│        ↓                                                     │

### 18.5 광고주를 위한 가치 제안

| **광고주 가치** | CPM (노출) | CPE (참여 품질) |

#### 포지셔닝 스테이트먼트

> **For** 시간을 가치 있게 활용하고 싶은 사람들
> **Who** 광고를 보고 실제로 돈을 벌고 싶지만 사기나 복잡함이 걱정되는
> **Advertise is** World ID 인증 기반 광고 참여 플랫폼
> **That** 퀴즈를 통해 검증된 참여에 대해 WLD로 즉시 보상
> **Unlike** 봇 문제와 출금 어려움이 있는 기존 Watch-to-Earn
> **Our product** 100% 실제 인간에게만 광고를 전달하고, 참여 품질을 증명합니다

### 23.8 기술 요구사항

#### 피드 구현

| 요구사항 | 설명 | 기술 |
|---------|------|------|
| **무한 스크롤** | 끊김 없는 콘텐츠 로딩 | Virtualized List |
| **비디오 플레이어** | 자동 재생, 음소거 토글 | HTML5 Video / HLS |
| **스와이프 제스처** | 상하 스와이프 인식 | Touch Events |
| **프리로딩** | 다음 콘텐츠 미리 로드 | Intersection Observer |
| **광고 추적** | 시청 시간, 완료 여부 | Custom Events |

#### 기프티콘 연동

| 요구사항 | 설명 | 방안 |
|---------|------|------|
| **기프티콘 API** | 발급/조회/취소 | 기프티쇼, 카카오선물하기 API |
| **결제 처리** | WLD → 기프티콘 | 내부 장부 + API 호출 |
| **재고 관리** | 실시간 재고 확인 | API 연동 |
| **배송** | 바코드/핀번호 전달 | 인앱 표시 + 카카오톡 |
