# Advertise 경쟁 환경 종합 보고서

> 작성일: 2025-02-04

## Executive Summary

Advertise는 **World ID 인증 기반 광고 시청 → 크립토 보상** 플랫폼입니다.
이 보고서는 직접 경쟁자, 간접 경쟁자, 대체 제품, 복합 제품을 분석하여 시장 포지셔닝 전략을 수립합니다.

---

## 1. 직접 경쟁자 (Direct Competitors)
> 같은 문제(광고 시청 보상), 같은 방식(크립토 지급)

| 기업 | 웹사이트 | 핵심 가치 제안 | 규모 |
|------|----------|----------------|------|
| **Brave Rewards** | [brave.com](https://brave.com/brave-rewards/) | 프라이버시 브라우저에서 광고 보고 BAT 토큰 획득 | 6,500만+ MAU |
| **Permission.io** | [permission.io](https://www.permission.ai/) | 브라우저 확장으로 광고 시청, ASK 토큰 획득 | 45만+ 유저 |
| **Verasity** | [verasity.io](https://verasity.io/) | Proof-of-View로 광고 사기 방지, VRA 토큰 보상 | - |
| **Cointiply** | [cointiply.com](https://cointiply.com/) | 광고/설문/비디오로 비트코인 획득 | 글로벌 |

### 상세 분석

#### Brave Rewards (BAT)
- **설립**: 2016년 (Brendan Eich, Firefox 공동창립자)
- **작동 방식**: 브라우저 내 옵트인 광고 → 광고 수익의 70%를 BAT로 유저에게
- **강점**: 검증된 비즈니스 모델, 대규모 유저 베이스
- **약점**: 브라우저 전환 필요, 광고당 보상이 매우 적음 (~$0.01-0.05)
- **기술**: Chromium 기반, Ethereum

#### Permission.io (ASK)
- **투자**: $50M+
- **작동 방식**: 크롬 확장으로 광고 시청 → ASK 토큰 획득
- **강점**: 데이터 주권 내러티브, AI 어시스턴트 통합
- **약점**: 유저 규모 작음, 토큰 유동성 낮음
- **기술**: Polygon 기반

## 2. 간접 경쟁자 (Indirect Competitors)
> 같은 문제(행동 → 보상), 다른 방식(광고 외 활동)

| 기업 | 웹사이트 | 핵심 가치 제안 | 규모 |
|------|----------|----------------|------|
| **Sweatcoin** | [sweatcoin.com](https://sweatcoin.com/) | 걷기로 SWEAT 토큰 획득 (Move-to-Earn) | 1.4억 유저 |
| **Galxe** | [galxe.com](https://www.galxe.com/) | Web3 퀘스트 완료하고 토큰/NFT 획득 | 3,200만 유저 |
| **Layer3** | [layer3.xyz](https://layer3.xyz/) | 온체인 퀘스트 + 교육 완료하고 토큰 획득 | 720만 유저 |
| **CARV** | [carv.io](https://carv.io/) | 게임 플레이 데이터로 보상 (Data-to-Earn) | 950만 유저 |
| **Telegram Tap-to-Earn** | Telegram 내 | 탭하여 크립토 획득 (Notcoin, Hamster Kombat 등) | 5억+ 방문자 |

### 상세 분석

#### Sweatcoin
- **비즈니스 모델**: 광고 + 구독($4.99/월) + 거래 수수료
- **2023년 매출**: $20M
- **강점**: 건강과 보상 결합, 비크립토 유저도 접근 가능
- **약점**: 봇/가짜 걸음 문제, Move-to-Earn Ponzi 논란

#### Galxe / Layer3
- **비즈니스 모델**: B2B SaaS (프로젝트가 퀘스트 비용 지불)
- **강점**: 대규모 파트너 네트워크 (6,700+ 파트너)
- **약점**: 에어드랍 헌터 위주, 일반인 진입장벽 높음

#### Telegram Tap-to-Earn (Notcoin, Hamster Kombat, TapSwap)
- **규모**: Hamster Kombat 5억+ 방문자, Notcoin 3,500만 플레이어
- **작동 방식**: 텔레그램 봇에서 탭 → TON 토큰 에어드랍
- **강점**: 엄청난 바이럴, 낮은 진입장벽
- **약점**: 지속가능성 의문, 토큰 가치 하락

## 4. 복합 제품 (Complementary Products)
> 함께 사용하면 시너지가 나는 제품

### 4.1 World 생태계 (필수 인프라)

| 제품 | 역할 | 우리 제품과의 관계 |
|------|------|-------------------|
| **World App** | 지갑 + 신원 인증 + 미니앱 플랫폼 | 우리 앱이 미니앱으로 동작 |
| **World ID** | Proof of Personhood | 봇 방지, 1인 1계정 보장 |
| **World Chain** | L2 블록체인 | 트랜잭션 처리 |
| **WLD 토큰** | 보상 통화 | 유저에게 지급하는 보상 |

### 4.2 시너지 가능 서비스

| 카테고리 | 예시 | 시너지 |
|----------|------|--------|
| **DeFi** | Uniswap, Aave | WLD 스테이킹/스왑으로 보상 극대화 |
| **NFT 마켓** | OpenSea, Blur | 광고 완료 배지를 NFT로 발행 |
| **Web3 소셜** | Lens, Farcaster | 소셜 공유로 추가 보상 |
| **결제** | World Pay, 기프트카드 | WLD를 실생활에서 사용 |

## 6. Gap 분석 & 기회

### 6.1 현재 시장의 미해결 문제

| Gap | 현황 | 영향도 |
|-----|------|--------|
| **봇/Sybil 공격** | 대부분 플랫폼이 취약 | 광고주 ROI 저하, 플랫폼 신뢰 하락 |
| **광고 참여 품질** | "보기만" 하면 보상 | 광고 효과 측정 불가 |
| **Web3 진입장벽** | 지갑 설정, 가스비 등 복잡 | 일반 유저 이탈 |
| **보상 가치** | 대부분 $0.01-0.05/광고 | 동기부여 부족 |
| **지속가능성** | 많은 프로젝트가 Ponzi 의심 | 규제 리스크, 유저 불신 |

### 6.2 Advertise 차별화 기회

| 차별화 축 | 우리 솔루션 | 기존 기업이 못한 이유 |
|-----------|-------------|----------------------|
| **Sybil 저항성** | World ID 1인 1계정 | 기술 시점 차이(2023년 등장), 통합 비용 |
| **참여 품질 검증** | 퀴즈로 광고 이해도 측정 | UX friction 우려, 하지만 보너스 XP로 해결 가능 |
| **낮은 진입장벽** | World App 미니앱 (설치 불필요) | 자체 플랫폼 lock-in 전략 |
| **신뢰할 수 있는 보상** | WLD (상장된 토큰, 실제 가치) | 자체 토큰은 가치 불확실 |

## 7. 전략적 포지셔닝

### 7.1 Value Proposition Canvas

```
┌─────────────────────────────────────────────────────────────┐
│  광고주 (Payer)                                              │
│  ─────────────────                                          │
│  Pain: 봇 트래픽, 광고 효과 측정 어려움                        │
│  Gain: 검증된 실제 인간에게만 노출, 이해도 메트릭              │
│                                                             │
│  우리 솔루션: World ID + Quiz = "증명된 관심"                 │
├─────────────────────────────────────────────────────────────┤
│  유저 (User)                                                 │
│  ─────────────────                                          │
│  Pain: 시간 낭비, 낮은 보상, 복잡한 절차                       │
│  Gain: 공정한 보상, 간단한 UX, 실제 가치 있는 토큰             │
│                                                             │
│  우리 솔루션: 미니앱 + WLD = "쉽고 가치 있는 보상"             │
└─────────────────────────────────────────────────────────────┘
```

### 7.2 경쟁 우위 요약

| vs 경쟁사 | 우리의 우위 |
|-----------|------------|
| vs Brave | 브라우저 전환 불필요, 퀴즈로 참여 품질 ↑ |
| vs Permission.io | World ID로 봇 방지, World 생태계 시너지 |
| vs Sweatcoin | 언제 어디서나 가능 (걷기 불필요) |
| vs Galxe/Layer3 | 일반인도 쉽게 접근 (에어드랍 헌터 X) |
| vs Tap-to-Earn | 지속가능한 비즈니스 모델 (광고 수익) |

## 8. 권장 액션

### 단기 (0-3개월)
1. **World App 미니앱 출시** - 생태계 선점
2. **광고주 3-5개 확보** - 초기 인벤토리
3. **퀴즈 정답률 데이터 수집** - 광고주에게 제공할 메트릭

### 중기 (3-6개월)
1. **광고주 대시보드 구축** - 셀프서브 광고 집행
2. **리퍼럴 시스템** - 유저 획득 비용 최소화
3. **프리미엄 광고 상품** - 높은 CPM 광고 카테고리 개발

### 장기 (6-12개월)
1. **멀티 리전 확장** - World ID 커버리지 따라 확장
2. **브랜드 파트너십** - Sweatcoin처럼 마켓플레이스 구축
3. **API/SDK** - 다른 앱에서도 Advertise 광고 노출
